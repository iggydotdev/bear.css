The Atomic Stack

The framework is built on a strict, 5-layer hierarchy. Everything traces back to a token, making it infinitely customizable.

  1. Level 0: Raw Values — Pure numbers, unitless (e.g. 4).
  2. Level 1: Primitives — Raw values given units (e.g. 0.25rem).
  3. Level 2: Tokens — Semantic names (e.g. --space-xs, --color-accent).
  4. Level 3: Properties — Tokens mapped to utility classes (e.g. .m-xs).
  5. Level 4: Compositions — Abstract layout patterns (e.g. .stack).

Customization

Because bear.css is built entirely on native CSS variables, you can customize the theme instantly without a build step. Just override the :root variables in your own stylesheet.

/* your-styles.css */
:root {
  /* Change the brand color */
  --color-accent: #8b5cf6;
  --color-accent-hover: #7c3aed;

  /* Increase border radii everywhere */
  --radius-sm: 0.5rem;
  --radius-md: 1rem;
}

Utility Classes

A quick reference to the naming conventions used in the framework.

Margin & Padding

Uses logical properties (block/inline) by default.

  • .m-{scale} / .p-{scale} (All sides)
  • .my-{scale} / .py-{scale} (Block/Vertical)
  • .mx-{scale} / .px-{scale} (Inline/Horizontal)
  • .mt-{scale}, .mb-{scale}

Typography

  • .text-{size} (xs to 3xl)
  • .font-{weight} (normal, medium, semibold, bold)
  • .leading-{size} (tight, base, loose)
  • .text-balance, .text-pretty

Layout Compositions

  • .stack (Vertical rhythm)
  • .cluster (Horizontal wrap)
  • .sidebar (Classic 2-col)
  • .center-xy (Dead center)
  • .cover (100dvh center)
